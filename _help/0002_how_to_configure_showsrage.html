--- name: How to configure ShowsRage? ---

Here are some examples to illustrate how to configure {{ site.github.project_title }}, under the Server section, based on your SickRage settings.

SickRage URL Use HTTPS? Hostname Port Number Path API URL
http://127.0.0.1/home/ clear 127.0.0.1 80 or empty api http://127.0.0.1/api/api_key/
http://127.0.0.1:8081/home/ clear 127.0.0.1 8081 api http://127.0.0.1:8081/api/api_key/
https://127.0.0.1:8083/folder/home/ check 127.0.0.1 8083 folder/api https://127.0.0.1:8083/folder/api/api_key/
http://sickrage.example.com/home/ clear sickrage.example.com 80 or empty api http://sickrage.example.com/api/api_key/
http://sickrage.example.com:8081/home/ clear sickrage.example.com 8081 api http://sickrage.example.com:8081/api/api_key/
https://sickrage.example.com:8083/folder/home/ check sickrage.example.com 8083 folder/api https://sickrage.example.com:8083/folder/api/api_key/

You also need to enter your API key in the API Key section. You can either enter it manually by clicking on API Key or enter your Username and Password and click on Get API Key. If you don't need a username and password to access SickRage, you can leave these fields empty.

info_outline Some users manage to get their API key, but the connection to SickRage is failling. This should be fixed in a future version.
In the meantime, if you encounter this issue, you can work around it by changing the port SickRage is running on.
